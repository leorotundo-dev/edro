# EDRO DA-IA Canon

## Tese
A IA visual da Edro não deve receber pedidos de estética solta. Ela deve receber problema, percepção, repertório, prova, estrutura e crítica com precisão operacional para virar campanha, não só imagem.

## Papel do operador
Quem opera o motor não está usando uma ferramenta de imagem. Está dirigindo um diretor de arte cego, rápido, obediente, literal demais e incapaz de julgar sozinho o que é estrategicamente certo.

## Fórmula de direção
`Função > sensação > linguagem > execução`

## 7 camadas de pilotagem
1. Pedir função, não imagem.
2. Definir o que a marca deve parecer e o que não pode parecer.
3. Dirigir repertório, não só adjetivos.
4. Trabalhar em etapas.
5. Usar a IA para explorar caminhos, não para decidir sozinha.
6. Dar briefing como diretor.
7. Criticar com precisão.

## Edro OS visual
1. Diagnóstico: qual problema real estamos resolvendo.
2. Verdade: o que o público sente, o mercado distorce e a marca pode provar.
3. Tensão: qual conflito energiza a comunicação.
4. Transformação: o que precisa mudar na percepção.
5. Território: que linguagem dá corpo à estratégia.
6. Big Idea: qual frase-mãe e qual imagem mental organizam tudo.
7. Prova: por que alguém acreditaria nisso.
8. Sistema de campanha: como isso vive em múltiplos canais.

## Matriz problema -> repertório
- Falta de credibilidade -> documental premium / operação real / institucional limpo
- Comoditização -> símbolo forte / graphic bold / sistema proprietário
- Baixa percepção de valor -> hero shot / premium acessível / composição refinada
- Falta de clareza -> design de informação / UX visual / didatismo gráfico
- Falta de conexão emocional -> cinema / humano observacional / cotidiano
- Precisa vender rápido -> varejo bem hierarquizado / contraste / resposta direta
- Falta de proximidade -> vernacular / cultura local / pessoas reais
- Precisa parecer inovador -> tech minimal / modularidade / interface language
- Precisa provar escala -> grande angular / mapas / estrutura / repetição
- Falta de memorabilidade -> símbolo / frame icônico / objeto dominante

## Stack híbrido de repertório
- `Input Layer`: transforma pedido solto em problema visual classificado.
- `Canon Engine`: usa Cannes Lions, D&AD e cases fortes para ensinar excelência estrutural.
- `Trend Engine`: usa Adobe Creative Trends, Behance e Pinterest Predicts para detectar sinais emergentes.
- `Edro Memory`: guarda só o ouro aprovado, aprendido ou reutilizável.
- `Edro Filter`: decide o que é estrutural, emergente útil, ruído ou incompatível.
- `Route Output`: devolve rotas seguras, diferenciadas, ousadas ou de laboratório.

## Mandamentos
- Dar problema, não só estética.
- Dar direção, não só adjetivo.
- Dar limites negativos com a mesma clareza dos positivos.
- Iterar por critério, não por gosto solto.
- Usar a IA para explorar e acelerar, não para pensar sozinha.
