# Modelos Comportamentais

Os dois frameworks que estruturam como o Motor pensa intervenção. Todo `BehaviorIntent` declara qual modelo primário está sendo usado.

---

## Fogg Behavior Model

**Equação:** Comportamento = Motivação × Habilidade × Prompt (todos precisam ser suficientes ao mesmo tempo)

**Aplicação no Motor:**

- **Motivação baixa?** → Não adianta aumentar o prompt. Trabalhar gatilhos de identidade e loss aversion primeiro.
- **Habilidade baixa?** → Simplificar a ação pedida. "Salve para ler depois" é mais fácil que "entre em contato".
- **Prompt ausente?** → CTA explícito com momento certo. Post sem CTA não converte mesmo com motivação e habilidade altas.

**Quando usar:** Diagnóstico de por que uma campanha não gerou o micro_behavior esperado. Perguntar: qual dos três estava baixo?

**Nível:** [padrão] — modelo consolidado, aplicável como framework de diagnóstico imediato.

---

## Hook Model (Nir Eyal)

**Ciclo:** Trigger → Action → Variable Reward → Investment

**Aplicação no Motor:**

- **Trigger:** O que faz a persona parar o scroll? (interno: dor/desejo latente; externo: headline, visual)
- **Action:** A ação mais simples possível (parar, ler, clicar — não pedir muito de uma vez)
- **Variable Reward:** O conteúdo entrega valor inesperado? Insight que eles não tinham?
- **Investment:** A persona faz algo que a prende ao próximo ciclo? (salvar, comentar, seguir)

**Quando usar:** Planejamento de sequências de conteúdo que constroem hábito de consumo com a marca.

**Nível:** [padrão] — modelo consolidado, aplicável como framework de planejamento de série.

---

## Qual modelo escolher?

| Objetivo da campanha | Modelo recomendado |
|---|---|
| Gerar uma ação específica única (request_demo, click) | Fogg |
| Construir audiência engajada ao longo do tempo | Hook |
| Diagnosticar campanha que não performou | Fogg (qual variável estava baixa?) |
| Planejar série de conteúdo de nurturing | Hook |
