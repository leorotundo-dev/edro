# MEMODROPS — GUIA DE DECISÃO

Este documento resume as diretrizes operacionais do **MemoDrops — conhecimento que fica. Gota a gota**. Use-o como checklist em produto, UX, IA, copy e roadmap.

## Regra zero
**Isso ajuda o conhecimento a ficar?**  
Se não for um “sim” claro, decline. Mesmo que seja bonito, tendência ou tecnicamente possível.

## 1. Produto
**Criaremos a feature se:**
- reduz esquecimento
- melhora retenção no tempo
- tira carga cognitiva do usuário
- diminui ansiedade por planejamento

**Não criaremos se:**
- só “completa” o produto
- aumenta configurações manuais
- existe apenas para comparação com concorrentes

> “Isso melhora memória ou só aumenta volume?”

## 2. UX/UI
**É correta se:**
- mantém ação principal clara
- não pressiona o usuário
- funciona com o usuário cansado
- respeita a forma líquido/orgânica das gotas

**Não é se:**
- lembra planilha/dash rígido
- usa vermelho/verde como julgamento
- exige atenção constante

> “Essa tela acolhe ou cobra?”

## 3. Copy
**Frases que cabem no MemoDrops:**
- explicam o porquê
- falam como parceiro
- normalizam erro
- reduzem culpa

**Fora do MemoDrops:**
- manda (“faça”, “complete”)
- julga (“você errou”)
- soa como professor/coach

> “Isso soa como quem entende memória?”

## 4. IA
**Usamos IA quando ela:**
- simplifica
- reorganiza
- aprende com erro
- tira carga mental

**Não usamos quando:**
- gera texto parrudo só para parecer inteligente
- responde genericamente
- explica sem contexto

> “A IA está pensando com o usuário ou falando sozinha?”

## 5. Gamificação
**É válida se:**
- incentiva retorno sem culpa
- aceita pausas
- reforça consistência leve

**Não se for:**
- punir ausência
- quebrar streaks de vez
- criar ansiedade de desempenho

> “Isso convida ou constrange?”

## 6. Cor e visual
**Escolhemos cores se:**
- regulam emoção
- ajudam foco prolongado
- respeitam roxo/lilás calmos
- mantêm o app silencioso

**Evita-se:**
- cores gritantes
- alertas sem risco real
- copiar vermelho/verde só porque “funciona”

> “Essa cor acalma ou excita sem necessidade?”

## 7. Roadmap
**Entram agora se:**
- desbloqueiam retenção
- fortalecem a experiência-base
- aumentam confiança no sistema

**Não entram se:**
- são só expansão de escopo
- aumentam complexidade sem ganho cognitivo
- resolvem problema que o usuário ainda não sente

> “Isso resolve esquecimento ou só adiciona coisa?”

## 8. Teste final obrigatório
> “Se o usuário estivesse cansado, ansioso e com pouco tempo, isso ajudaria ou atrapalharia?”  
Se atrapalha, corta. MemoDrops funciona mesmo no pior dia.

## Síntese única (colar na parede)
**MemoDrops é um sistema que pensa junto com o usuário. Nunca contra ele.**  
Se algo quebrar isso, não é MemoDrops.
